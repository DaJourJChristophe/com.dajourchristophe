# Security Policy

If you discover a security issue in this project, please report it privately to:

- dajour.christophe@gmail.com

Please do not open a public issue for vulnerabilities that could expose users, infrastructure, credentials, or deployment details.

## What to include

When reporting a security issue, include:
- a clear description of the issue
- impact and affected area
- reproduction steps when possible
- any suggested mitigation

## Response expectations

The goal is to acknowledge reports promptly, validate the issue, and address confirmed problems as quickly as practical.

## Scope

Security-relevant concerns may include:
- dependency vulnerabilities with real project impact
- server misconfiguration
- sensitive data exposure
- authentication or authorization issues if introduced later
- unsafe handling of user-controlled input
